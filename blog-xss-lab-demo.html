    XSS Exploit Walkthrough for Security Blog Layout Testing

    This article is a controlled lab-only walkthrough to demonstrate how security writeups can present evidence clearly. It covers Reflected XSS, Stored XSS, and DOM XSS with safe remediation notes.

    Run these steps only on systems you own or have explicit permission to test.

    1) Vulnerable Lab Setup (Example)

    Typical anti-pattern in test apps: injecting user input directly into HTML without encoding.

    // vulnerable rendering example (lab only)
app.get("/search", (req, res) => {
  const q = req.query.q || "";
  res.send(`<h2>Result for: ${q}</h2>`);
});

    2) Reflected XSS Demo

    1. Identify reflected parameter: /search?q=...
    2. Inject payload into URL and open in browser.
    3. Observe JavaScript execution in response context.
    GET /search?q=%3Cimg%20src%3Dx%20onerror%3Dalert('xss-lab')%3E

    3) Stored XSS Demo

    Stored XSS appears when comment/profile fields are saved and later rendered unsafely for other users.

    <!-- payload used in lab comment field -->
<script>alert("stored-xss-lab")</script>

    Verify with two accounts: attacker submits payload, victim opens page and executes script.

    4) DOM XSS Demo

    DOM XSS often appears when code writes untrusted data to innerHTML.

    // vulnerable client-side snippet
const hash = decodeURIComponent(location.hash.slice(1));
document.getElementById("msg").innerHTML = hash;
    https://lab.local/page#<img src=x onerror=alert('dom-xss-lab')>

    5) Security Fix Pattern

    Vulnerability Root Cause Recommended Fix
    Reflected XSS Unsafely rendering query input Context-aware output encoding
    Stored XSS No sanitization before render Server-side validation + output encoding + CSP
    DOM XSS Using innerHTML with untrusted data Use textContent or trusted sanitization (e.g. DOMPurify) 
    // safer client-side rendering
const hash = decodeURIComponent(location.hash.slice(1));
document.getElementById("msg").textContent = hash;

    For report quality, include exploit precondition, reproduction steps, impacted roles, and retest results. Short inline sample: CSP: default-src 'self'; script-src 'self'.
