Lecture 2: Android Tools and ADB Basics

This guide covers the core setup steps for Android testing with ADB on an emulator.

1. Enable Developer Mode and USB Debugging

On the emulator, tap the build/version field around 7-8 times to unlock Developer Options.

Open Developer options, enable USB debugging, and allow ADB communication from your host machine.

2. Validate ADB Connectivity

Check whether the emulator is visible to ADB:

adb devices

Restart ADB daemon as root on the emulator:

adb devices
adb root

If you see restarting adbd as root, the command succeeded.

3. Install APK via ADB

Sample APK: InsecureBankv2.apk

adb install /path/to/apkfile

Run this from the APK directory or provide a full absolute path.

4. Essential ADB Commands

Open an interactive shell on the device:

adb shell

View logs:

adb logcat

Upload a file from host to device:

adb push </path/to/file/on/host> </path/to/file/on/device>

Download a file from device to host:

adb pull </path/to/file/on/device> </path/to/file/on/host>
